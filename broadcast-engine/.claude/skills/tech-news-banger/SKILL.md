# Tech News Banger - 科技新闻"炸锅体"口播稿改写技能

## Skill简介

把科技/医学/AI/机器人类新闻改写成短视频口播稿，让复杂技术讲成普通人能秒懂的话，开头3秒抓人，结尾有升华，接近成熟科技号可直接开录的水平。

**核心理念**: 不是写新闻稿，是做"认知按摩"——把复杂变简单，把平淡变炸裂，把知道变震惊。

---

## 适用场景

- 需要把科技新闻做成短视频口播稿
- 需要把技术内容讲得接地气、有网感
- 需要提升内容的信息密度和情绪张力
- 需要产出可直接录制的成熟稿件

---

## 执行步骤

### Step 1: 分析输入素材
读取原始新闻，提取：
- 核心事件（一句话）
- 关键技术/产品
- 关键数据（数字要精准）
- 突破点/创新点
- 意义/影响

### Step 2: 识别爆点
判断"炸点"在哪里：
- 首次/突破类（全球首个、首次）
- 颠覆认知类
- 数据炸裂类
- 改变生活类
- 争议/冲突类

### Step 3: 找比喻
找到普通人能理解的类比：
- 用生活场景解释技术
- 用熟悉事物类比新事物
- 数字要具象化

### Step 4: 按模板写作
按以下结构填充内容：

```
[开场hook - 3秒]     ← 颠覆认知/震撼数据/场景代入
[背景铺垫 - 10秒]    ← 简述这是什么，建立框架
[核心炸点 - 20秒]    ← 最重要的信息，要有数据
[细节支撑 - 15秒]    ← 技术简化版+具体例子
[认知升华 - 10秒]    ← 意味着什么，拔高价值
[互动收尾 - 2秒]     ← 提高互动率
```

### Step 5: 自检优化
- [ ] 开头3秒能不能抓住人？
- [ ] 有没有具体数字？
- [ ] 普通人能听懂吗？
- [ ] 有没有让人"卧槽"的点？
- [ ] 读起来顺不顺？（必须朗读测试）
- [ ] 结尾有没有升华？

---

## 核心风格要点

### 要用的
- **短句**：每句不超过15字
- **动词**：颠覆、炸裂、碾压、秒杀
- **数字**：具体数字比模糊描述好10倍
- **对比**："不是XX，是XX"
- **感叹号**：情绪担当，舍得用

### 不要用的
- "非常"、"极其"、"相当"（不炸）
- "研究表明"、"专家表示"（太教科书）
- 连续三个以上的长句（会喘不过气）
- "近日"、"据报道"（上来就用）

### 节奏口诀
> 震惊开场 → 悬念递进 → 细节轰炸 → 认知刷新 → 升华收尾
> 3秒内 → 15秒内 → 30秒内 → 45秒内 → 60秒内
